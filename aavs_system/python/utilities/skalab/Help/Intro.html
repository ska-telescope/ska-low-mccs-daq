SKALAB

 

The idea of the SKA in LAB project (SKALAB) is to provide quick access to SKA-Low devices in the lab without the use of complex software frameworks, but, it uses the same communication interfaces to ensure compatibility over time.

 

In a very preliminary version, was born in 2016 when the prototyping of the first TPM (Tile Processing Module) just ended and the first collection of python scripts were available. The first GUI was just a wrapper of low level functions (access-layer) providing basic functionality for programming FPGAs, initializing the board, acquiring ADC data, and monitoring power consumption and temperatures.

 

With the implementation of the 2 Aperture Array Verification Systems (AAVS) and the lessons learned, the GUI started to include other features such as: station status checks, antenna spectra monitoring, levels equalization, antenna mapping, but also, an environment able to plot the recorded quantities in order to do some basic data analysys.

 

Then, with the new TPM design and the TPM Subrack it started to provide also easy acces to the Subrack Management Board (SMB).

 

Now SKALAB is a macro container that allows to use stand alone modules initialized by using custom user profiles. Modules are shown in a Tab Widget, each module provides the HELP page as user manual.

Profiles

 

Both SKALAB and modules have their own profile files.

 

Profiles will be stored in the user path (~/.skalab/<PROFILENAME>/<MODULENAME>.ini) and will not be lost after software updates. The predefined profiles ("Default") will be automatically generated and the user can modify [SAVE] or create new configurations [SAVE AS...] by modifying the current profile within SKALAB or within each module.

 

Several profiles can be saved to easy recall [LOAD] the set of configurations used like: configuration file, axis preferences, spectral resolution, view mode (spectra, spectrogram, power, selected polarization). Obsolete profiles may be deleted [DELETE].

 

SKALAB has also a main initialization file stored in "~ / .skalab/skalab.ini" that is used to remember [AUTOSAVE] the settings when the application is closed, and to automatically load [AUTOLOAD] that configuration.

 

Autosave on Exit

if checked the last used setup will be saved in the current profile without asking for confirmation.

 

AutoLoad on Program Start

if checked when the program starts the last profile used will be loaded.
